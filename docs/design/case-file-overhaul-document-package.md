# Case File Overhaul — Document Package Structure

**Status:** Agreed

## The Case File Package

A complete case file consists of the following documents:

| # | Document | Audience | Format | Notes |
|---|---|---|---|---|
| 1 | **Case File** (mission brief) | Players | Fillable form, 1 page | In-world document. Only what the Covenant would share with field agents: relic surface description, region, timeline pressure, known organizations, starting NPC dossiers and regional contacts. No secrets, no containment truths, no milestone details. |
| 2 | **Case Brief** (mystery statement + DA notes) | DA | 1 page | The "real story" summary. What's actually going on, who's really involved, what the answer is. DA-only anchor document. |
| 3 | **Operations Board** | DA | Single sheet | Phases row (doubles as relic timeline) + organization countdown rows. See operations-board design doc. |
| 4 | **Organization Reference** | DA | Single sheet | All case-specific organizations listed with O#M# milestone descriptions and cross-link instructions. DA scans this when crossing out milestone squares on the Operations Board. |
| 5 | **Relic Sheet** | DA | 1 page | Full artifact profile: tier, category, what it does, activation condition, mechanical effects, fracture table, containment profile, and containment truth checklist. All artifact knowledge in one place. |
| 6 | **Location Pages** | DA | 1 page each | One per location. Primary play-aid — the document in front of the DA when players arrive somewhere. Fields: name, description, availability, NPCs present (milestone-driven), information available, organizations present, positive/negative results, milestone changes. |
| 7 | **NPC Cards** | DA | Playing-card size | All agreed fields. DA pulls relevant cards and places next to the active location page. |
| 8 | **Information Cards** | DA → Players | Playing-card size, double-sided | **Front (player-facing):** what the agents learned. **Back (DA reference):** source locations, source NPCs, HQ fallback phase, type (containment truth vs. supporting intel). Players physically receive cards when information is discovered — becomes tangible progress tracker. |

## Pre-Made Faction Dossiers (Core Rules / GM Toolkit)

Reusable organization reference sheets for recurring factions. Not per-case documents — part of the core rules or GM toolkit section. Each covers:

- Faction name, goals, and operational style
- Typical resources and reach
- Common operative types
- Blank milestone template the DA fills in for a specific case

## Typical Package Size

For a single-session case with 5-6 locations and 6-8 NPCs:
- 4 standard sheets (Case File, Case Brief, Operations Board, Organization Reference)
- 1 Relic Sheet
- 5-6 Location Pages
- ~8 NPC cards
- ~10-12 Information cards
- **Total: ~10-12 printed pages + ~20 cards**

## Key Design Decisions

- **Information cards go to players.** Double-sided design keeps DA reference hidden. Players spread out discovered cards and look for connections — tangible investigation tracker.
- **Containment truths live on the Relic Sheet**, not on information cards. Keeps all artifact knowledge centralized for the DA.
- **Organization Reference is a single sheet**, not per-org cards. DA needs to scan milestone text quickly when crossing out Operations Board squares; one page is faster than flipping through cards.
- **Pre-made faction dossiers reduce prep** for recurring organizations. DA customizes milestones per case but doesn't need to rebuild faction background each time.
