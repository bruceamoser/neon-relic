# Case File Overhaul — Operations Board Design

**Status:** Agreed

## Terminology Change

**"Front" is renamed to "Organization."** Every pressure source on the board traces back to an organized group of people — the Compact, the Church, the police precinct, the news crew, the crime syndicate. The label "Organization" is clearer for new DAs and maps to real-world understanding. Shorthand: **O#M#** (Organization 1, Milestone 1).

## Layout

- **14 columns**, all counting down left to right (14 → 1).
- Column 14 = case start. Column 1 = catastrophe (relic activation, sale, city exposure, etc.).
- Frame it as **"Days until catastrophe"** — column 1 has concrete narrative weight.

### Row 1: Phases (also the Relic Timeline)

- Each day-cell is quartered with an X, creating 4 quadrants: Morning / Day / Evening / Night.
- Fill in quadrants as shifts pass.
- **Relic milestones are keyed to specific phase numbers on this row.** The phases row is both the clock and the artifact's escalation track. When the DA fills in a shift quadrant that crosses a relic milestone phase, the artifact event triggers.
- At Phase 1, catastrophe occurs — the relic's worst-case outcome fires.
- Relic milestone descriptions live in the Case Brief, keyed to phase numbers.
- Replaces both the separate shift handout and the separate Relic Timeline row.

### Organization Rows

- Each row has: **Name** | **Active checkbox** | **14 squares** (countdown).
- DA fills from column 14 down to the organization's starting value at case start.
- Milestone squares are labeled with **O#M#** shorthand (e.g., O1M1 = Organization 1, Milestone 1).
- Escalation = cross out the next square moving toward column 1.
- Milestone descriptions live in the organization reference text, not on the grid.

### Cross-Links

- No arrows or special grid notation.
- Milestone text contains the instructions: *"O2M3: A wounded thief surfaces. Color in 2 Police squares and 1 Compact square."*
- DA reads the milestone description and executes.

### Dormant Organizations

- Pre-printed on the chart with an **Active** column/checkbox between the name and column 14.
- Row is inert until marked active.
- Chart stays stable — no adding rows mid-session.

## Design Principles

- Everything counts down left to right. Days count down, organizations count down. Same direction, same visual metaphor.
- The phases row serves double duty: shift clock + relic escalation. No separate relic row needed.
- Dead space on low-starting organizations (e.g., Heist Crew starting at 6 = columns 14–7 filled) visually communicates impending doom. That's a feature.
- Most cases target a single session (some 2-session). Campaigns are a series of case files.
- Recommend a maximum of ~8 organization rows.

## Replaces

- Separate Relic Timeline tracking (merged into phases row)
- Separate Front/Organization milestone lists
- Separate Shift Timeline / 10-day handout
- The original Steps 4, 5, and 8 from the old 10-step process
